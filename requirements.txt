# Layer 1 (Breeding Risk) has ZERO third-party dependencies -- it runs on a
# bare Python 3.9+ install using only the standard library (urllib, json,
# datetime). This keeps the free GitHub Actions cron trivial to set up.
#
# Layer 2 (Fever Signal) also runs dependency-free on its DEFAULT mock/sample
# provider. The real providers add deps only if you actually select them:
#   pytrends        # --provider pytrends  (unofficial, archived; imported lazily)
#   # --provider serpapi needs no pip install, only a SERPAPI_KEY env var (stdlib HTTP)
#
# Layer 3 (Confirmed Cases) also runs dependency-free on its DEFAULT `sample`
# (synthetic) source. The REAL sources -- `fixture` (parse a saved IDSP PDF) and
# `live` (download + parse the newest weekly PDF) -- need pdfplumber to read the
# outbreak tables. Discovery + download stay stdlib-only (urllib via httputil),
# so pdfplumber is the single heavy dependency, imported lazily so importing the
# package never fails when it is absent.
pdfplumber>=0.11   # Layer 3: parse IDSP weekly outbreak PDFs (fixture / live sources)
